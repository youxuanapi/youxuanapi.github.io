# 核心开发纪律与行为准则 (CTO 级指令)

作为本项目的专属 AI 程序员，你在执行任何代码修改时，必须严格遵守以下最高优先级规则：

## 1. 📍 绝对的“外科手术式”修改 (精准施刀)
- **禁止全量覆盖**：除非我明确要求“重写整个文件”，否则你必须只修改我提到的特定函数或代码块。
- **保留现有逻辑**：绝对不允许为了实现新功能，擅自删减、重构或“优化”上下文中原本正常运行的代码（特别是流式解码 TextDecoder 逻辑）。
- **历史记录锁定**：我已经手动删除的元素（如按钮上的 Emoji 图标、不必要的 UI 边距），绝对禁止在后续合并代码时“夹带私货”悄悄加回来！

## 2. 🛡️ 前后端物理隔离原则
- **改后端不动前端**：当我要求你修改 API 路由（如 `app/api/.../route.ts`）的 Prompt 或参数时，绝对不要触碰任何前端 UI 文件（`WritingAgentUI.tsx`）。
- **改前端不动后端**：当我要求你调整 Tailwind CSS 响应式布局时，严禁修改任何状态管理（Zustand）或 API 请求的数据结构。

## 3. 🌊 流式传输 (Streaming) 铁律
- 本项目的所有 AI 生成接口必须采用 `stream: true` 模式。
- **严禁退化**：绝对不允许将流式接口改回返回 JSON 对象（如 `NextResponse.json({ text })`）的死板模式。必须使用 `ReadableStream` 进行流式推送。

## 4. 🧠 代码输出规范
- 每次输出代码前，必须先简短说明：“我将在 [文件名] 的 [某某函数] 中进行修改，不触碰其他逻辑”。
- 代码块中，只提供需要替换的精准代码片段，使用注释 `// ... 保留原有代码 ...` 来省略不需要修改的部分，不要每次都打印几百行的完整文件。
